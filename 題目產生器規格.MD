# VCF 題目產生器規格

文件版本：0.9  
文件日期：2026-07-19  
目前實作：可指定產生 1～10 步 VCF

## 1. VCF 步數

VCF 步數是**最終連五之前，攻方實際落子的手數**。

```text
攻方死四 → 守方擋 → 攻方活四 → 守方擋 → 攻方五連
```

以上為 2 步 VCF：連五前攻方實際下了「死四」與「活四」兩手；最後連五那手不計入。

同一手即使同時形成四四，也仍然只算 1 步，因為攻方實際只落了一手。程式必須重播手順並計算連五前的攻方落子數，不得依形成幾個四來加總。

## 2. N 點

N 點分為兩個獨立標記：

- `NO_BLACK`：黑方不能使用。
- `NO_WHITE`：白方不能使用。

同一點可同時具有兩個標記。判斷時依實際要放入棋子的顏色決定。

顯示方式：

- 綠色方塊：雙方 N 點。
- 紅框黑色方塊：黑方 N 點。
- 紅框白色方塊：白方 N 點。
- N 點方塊的幾何中心必須與棋盤交點完全重合。

## 3. 初始材料

初始材料包含活三與死四兩類模板。

### 3.1 活三

```text
留 留 攻 攻 攻 留
留 攻 留 攻 攻 留
```

- 三個「攻」為攻方棋子，皆可作 A。
- 三個「留」為雙方 N 點。
- 中間的「留」為形成活四的攻擊點。
- 最外側兩個「留」為後續五連點。

初始活三接上外層死四後，正常形成 2 步基礎：A 死四為第 1 步，回到初始活三形成活四為第 2 步。

### 3.2 死四

```text
X 攻 攻 攻 攻 留
攻 攻 攻 留 攻
攻 攻 留 攻 攻
```

死四一：

- X 必須為盤端或守方棋子。
- X 在盤內時為攻方 N 點。
- 靠近 X 的攻方棋子不能作 A。

死四二、三：

- 最外側兩顆攻方棋子不能作 A。
- 靠近內部「留」的攻方棋子可作 A。

所有「留」均為雙方 N 點。

初始死四接上一層新死四時，A 一手會同時完成兩個四，形成 1 步 VCF。若目標為 2 步以上，死四基礎必須比活三基礎多增加一層死四，才能達到相同步數。

有禁手且黑方進攻時，A 形成四四屬於禁手，因此死四基礎不可用；1 步模式會直接提示無合法題目，2 步以上則只使用活三基礎。

## 4. 新增死四模板

```text
1. X 五 攻 攻 攻 攻 X
2. X 攻 五 攻 攻 攻 X
3. X 攻 攻 五 攻 攻 X
```

- `攻`：攻方棋子。
- `五`：死四形成後守方唯一被迫防守的位置。
- `X`：模板外側位置。
- A 必須是既有攻方棋子，並與模板的一個「攻」位置重合。
- 新死四不預先限制方向，可與初始材料或前一層材料同方向；是否成立交由 N 點、棋型與完整 VCF 驗證決定。

## 5. 模板相容規則

### 攻方棋子位置

可沿用既有攻方棋，或在不是攻方 N 點的空點新增攻方棋。不能覆蓋守方棋。

### A 點

A 必須是原盤面既有攻方棋子，不得是初始材料標記為禁止 A 的棋子。完整套入死四後再移除 A，成為新題目的第一手。

### 五點

- 不能是攻方 N 點。
- 不能已有攻方棋子。
- 可為空點或原本的守方棋子。
- 原本是守方棋時，生成時先移除，VCF 過程中守方會下回。

### X 點

X 可為空點、守方棋子、任一種 N 點或盤端；唯一基本限制是不能已有攻方棋子。

### 方向

產生器不以「必須轉向」作為人工限制，四個方向都會枚舉。

- 初始活三：同方向模板若碰到「留」的攻方 N 點，會由相容規則自然淘汰；若未碰到且完整驗證成立，仍可保留。
- 初始死四：新死四可與初始死四同方向。此時 A 下回必須形成單線四四，且該線恰有兩個不同的五點，分別是初始死四的「留」與新模板的「五」。該四四只算 1 步。
- 後續延伸：每新增一層，完整路線的攻方落子數必須恰好比前一層增加 1。

### 新死四產生活三

套入完整死四並移除 A 後，掃描新死四範圍的所有盤內空點。只要攻方在某點合法落子後能於該方向形成活四，即判定存在活三。

若有活三，只能在左右 X 補守方棋；依序試左、右、兩端，補後重新掃描。仍有活三則淘汰候選。需要新增守方棋的 X 必須為空點且不能是守方 N 點。

## 6. 指定步數生成流程

使用者可指定 1～10 步。

### 6.1 選擇 1 步

1. 只枚舉初始死四。
2. 在初始死四外再加一層死四並移除 A。
3. A 下回必須合法形成四四，因此是 1 步 VCF。
4. 不得存在 0 步或其他更短的取勝路線。
5. 有禁手且黑方進攻時，由於四四是禁手，此模式沒有合法候選。

### 6.2 選擇 2～10 步

1. 建立目前規則下可用的活三與死四初始材料。
2. 活三與死四都可用時，先在兩種類別間等機率選擇，再於該類別內依初始材料權重抽選位置。
3. 活三接一層死四，先建立 2 步基礎。
4. 死四接一層死四，先建立 1 步基礎。
5. 若基礎步數尚未達到指定步數，繼續從目前盤面選擇既有攻方棋作 A，每次增加一層死四。
6. 每成功增加一層，連五前攻方實際落子數必須正好增加 1。
7. 重複延伸，直到達到指定步數。

因此相同目標步數下：

```text
活三路線新增死四層數 = 目標步數 - 1
死四路線新增死四層數 = 目標步數
```

例如產生 2 步：

```text
活三初始 + 1 層死四 = 2 步
死四初始 + 2 層死四 = 2 步
```

某一候選無法繼續時，程式會回溯嘗試同一層的其他候選；整個基礎都無法延伸時，才重新建立新的基礎題目。沒有設定每層最大嘗試次數，使用者可隨時停止。

## 7. 標準完成盤面

延伸判斷不比較手順順序，只比較黑白棋位置。

標準完成盤面：

- 終局為四四：記錄到四四形成後的盤面。
- 終局為活四：記錄到攻方下出活四之前的盤面。
- 其他終局：記錄到終局棋型完成的盤面。

新增一層後，新標準完成盤面與前一層相比，只能永久增加：

- 本層新增的攻方棋子。
- 本層為封閉活三而在 X 新增的守方棋子。

以下不算差異：

- A 在生成時被移除，VCF 過程中重新下回。
- 五點原有守方棋被移除，VCF 過程中重新下回。

## 8. 其他 VCF

每層均使用現有多組 VCF 功能：

1. 必須存在符合完成盤面差異的目標 VCF。
2. 基礎題必須符合該材料的步數：死四為 1 步，活三為 2 步。
3. 後續延伸的目標步數必須等於前一層步數加 1。
4. 若任何其他 VCF 比本層目標短，淘汰候選。
5. 等長或更長的其他 VCF 允許。
6. 不額外保存其他組的分支數或數量。

## 9. 隨機性與自訂加成

候選先完成列舉、基本棋型篩選與去重，再依權重建立一份加權隨機順序。候選排在第一位的機率為：

```text
候選抽中機率 = 該候選權重 ÷ 全部候選權重總和
```

因此權重為 100 的候選，相對於權重為 1 的未加成候選，排在第一位的機率高 100 倍。兩項輸入範圍均為 0～100%。

### 9.1 百分比轉換

設定百分比先轉換成最高權重增量：

```text
最高權重增量 = 99 × 設定百分比 ÷ 100
```

- 設定 0%：最高增量為 0，完整加成仍為 1 倍。
- 設定 50%：最高增量為 49.5，完整加成為 50.5 倍。
- 設定 100%：最高增量為 99，完整加成為 100 倍。

### 9.2 沿用攻子每顆加成

- 預設：10%。
- A 本身不計入沿用數；只計算新死四另外三個攻位中，直接沿用原盤面攻方棋的數量。
- 每沿用一顆，就增加一次「沿用攻子最高權重增量」。
- 例如設定 100%：沿用 0 顆時權重增加 0；沿用 1 顆增加 99；沿用 2 顆增加 198；沿用 3 顆增加 297。

### 9.3 朝天元綜合加成

- 預設：15%。
- 先以新死四四個「攻」位置（包含 A）計算攻子平均位置。
- 朝向分數：比較「A 指向攻子平均位置」與「A 指向天元」的夾角，朝向天元為 1，垂直或背向為 0。
- 距離分數：攻子平均位置在天元為 1，離天元越遠越接近 0；以天元到棋盤角落的距離作正規化。
- 綜合分數為：

```text
朝天元綜合分數 = 朝向分數 × 距離分數
```

完整朝向天元且攻子平均位置在天元時，綜合分數為 1；任何一項為 0 時不取得朝天元加成。

### 9.4 最終權重

```text
沿用增量 = 沿用攻子數 × 99 × 沿用設定百分比 ÷ 100
朝天元增量 = 朝天元綜合分數 × 99 × 朝天元設定百分比 ÷ 100

候選權重 = 1 + 沿用增量 + 朝天元增量
```

兩項加成採相加而不是相乘。例如沿用一顆與朝天元綜合分數滿分，兩項都設為 100% 時，候選權重為 `1 + 99 + 99 = 199`。

加成只影響候選被優先嘗試的機率，不會放寬 N 點、禁手、棋型、步數或較短 VCF 的驗證規則。

## 10. 非要求與後續項目

- 不要求盤面能由正常黑白交替對局走出來。
- 不自動補棋封鎖其他較短 VCF；遇到較短組直接淘汰。
- 目前不使用 Worker Pool 平行驗證。
- 後續仍可改用既有 VCF 程式重新計算更完整的 N 點。
- 必要時可加入盤面快取與平行驗證。
