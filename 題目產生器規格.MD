# VCF 題目產生器規格

文件版本：0.3  
文件日期：2026-07-19  
目前實作：固定產生 2 步 VCF 的試作版

## 1. VCF 步數定義

VCF 步數是攻方在最後連五之前，連續下出「四」的數量。

```text
攻方死四 → 守方擋 → 攻方活四 → 守方擋 → 攻方五連
```

以上為 2 步 VCF。

若同一手形成四四，該手同時產生兩個四，因此步數加 2。步數不得直接用完整手順陣列長度代替，必須重播手順並依棋型計算。

## 2. N 點

N 點只分為兩種獨立標記：

- `NO_BLACK`：黑方不能使用。
- `NO_WHITE`：白方不能使用。

同一點可同時具有兩種標記。判斷時依準備放入棋子的顏色決定：

```text
放黑棋 → 檢查 NO_BLACK
放白棋 → 檢查 NO_WHITE
```

## 3. 第一版初始材料

初始材料會先在「活三」與「死四」兩類中隨機選擇，再於該類內隨機選棋型、方向與位置。

### 3.1 活三

```text
留 留 攻 攻 攻 留
留 攻 留 攻 攻 留
```

- 三個「攻」為攻方棋子，皆可作 A。
- 三個「留」同時標記 `NO_BLACK` 與 `NO_WHITE`。
- 中間的「留」為形成活四的攻擊點。
- 最外側兩個「留」為後續五連點。

### 3.2 死四一

```text
X 攻 攻 攻 攻 留
```

- `X` 必須為盤端或守方棋子。
- X 在盤內時標記為攻方 N 點。
- 「留」為雙方 N 點。
- 靠近 X 的第一顆攻方棋子不能作 A。
- 其餘三顆攻方棋子可作 A。

### 3.3 死四二

```text
攻 攻 攻 留 攻
```

- 「留」為雙方 N 點。
- 最左與最右兩顆攻方棋子不能作 A。
- 中間兩顆攻方棋子可作 A。

### 3.4 死四三

```text
攻 攻 留 攻 攻
```

- 「留」為雙方 N 點。
- 最左與最右兩顆攻方棋子不能作 A。
- 靠近「留」的兩顆攻方棋子可作 A。

### 3.5 禁手限制

初始死四與新死四共用 A。A 下回時會同時形成原死四與新死四，也就是四四。

- 白方攻擊：允許。
- 無禁手模式的黑方攻擊：允許。
- 有禁手模式的黑方攻擊：四四為禁手，因此不使用初始死四，只使用初始活三。

## 4. 新增死四模板

```text
1. X 五 攻 攻 攻 攻 X
2. X 攻 五 攻 攻 攻 X
3. X 攻 攻 五 攻 攻 X
```

- `攻`：攻方棋子。
- `五`：攻方形成死四後，守方唯一被迫防守的位置。
- `X`：模板外側位置。
- A 必須是原盤面既有攻方棋子，並與模板中的任一 `攻` 位置重合。

## 5. 模板相容規則

### 5.1 攻方棋子位置

可使用：

- 原盤面已有的攻方棋子。
- 空點，且不是攻方 N 點；此時新增攻方棋子。

不可使用：

- 守方棋子。
- 攻方 N 點上的空點。

### 5.2 A 點

A 必須：

- 原盤面已是攻方棋子。
- 與新死四模板中的一個 `攻` 位置重合。
- 不屬於初始死四標記的禁止 A 棋子。
- 完整套入死四後再移除，成為題目的第一手。

### 5.3 五點

五點：

- 不能是攻方 N 點。
- 不能已有攻方棋子。
- 可為空點。
- 可原本是守方棋子；生成時先移除，VCF 過程中守方可下回。

### 5.4 X 點

X 可以是空點、守方棋子、任一種 N 點或盤端；唯一基本限制是不能已有攻方棋子。

### 5.5 移除 A 後的活三

套入完整死四並移除 A 後，必須檢查新死四方向是否留下合法活三。

- 掃描新死四模板涵蓋的所有盤內位置。
- 對每個空點試下攻方棋。
- 若該手合法且在新死四方向形成活四，表示材料仍是活三。
- 有禁手且攻方為黑方時，禁手延伸不算合法活四點。

若形成活三，只能在兩端 X 補守方棋，可補左、右或兩端。補完後重新掃描；仍有活三則淘汰。

需要在 X 新增守方棋時：

- X 必須為空點。
- X 不能是守方 N 點。

盤端或已有守方棋子的 X 視為已封閉。

## 6. 隨機生成

1. 隨機選初始活三或初始死四；兩類都有候選時，類別機率相同。
2. 在該類內加權隨機選方向與位置。
3. 從允許的既有攻方棋子中選 A。
4. 從其他方向套入三種新死四模板之一。
5. 移除 A。
6. 檢查並封閉新死四產生的活三。
7. 使用現有多組 VCF 功能完整驗證。

候選以加權隨機排列，不直接選最高分：

- 沿用更多既有攻方棋子可有小幅加成。
- 朝天元方向生成可有小幅加成。
- 兩項加成都可關閉。

不設定每層最大嘗試次數；使用者可隨時停止。

## 7. 第一版延伸驗證

### 7.1 初始活三

目標路線須符合：

1. VCF 步數為 2。
2. 第一手為新 A。
3. 第二手為新死四的五點。
4. 第三手為初始活三形成活四的位置。
5. 後續防守與五連點符合初始活三兩端 N 點。

### 7.2 初始死四

目標路線須符合：

1. VCF 步數為 2。
2. 第一手為新 A。
3. A 下回後，在初始死四方向形成死四。
4. A 下回後，在新生成方向形成死四。
5. 兩個死四的唯一防點分別為初始「留」與新模板「五」。
6. A 的棋型必須被判定為四四，步數一次加 2。

## 8. 其他 VCF 的處理

1. 目標 VCF 必須成功成立，否則放棄候選。
2. 若任何其他 VCF 比目標 VCF 短，放棄候選。
3. 等長或更長的其他 VCF 允許。
4. 不額外保存其他組的分支數或數量。

## 9. 非要求

- 不要求盤面能由正常黑白交替對局走出來。
- 不自動補棋封鎖其他較短 VCF；遇到較短組直接淘汰。
- 第一版不使用 Worker Pool 平行驗證。

## 10. 後續項目

- 從現有 VCF 路線重新計算完整 N 點。
- 任意目標步數。
- 標準完成盤面比較與多層延伸。
- 必要時加入候選快取與平行驗證。
