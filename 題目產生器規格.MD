# VCF 題目產生器規格

文件版本：0.1  
文件日期：2026-07-18  
目前實作：固定產生 2 步 VCF 的試作版

## 1. VCF 步數定義

VCF 步數是攻方在最後連五之前，連續下出「四」的次數。

例如：

```text
攻方死四 → 守方擋 → 攻方活四 → 守方擋 → 攻方五連
```

以上為 2 步 VCF。

步數不得直接用完整手順陣列長度代替，應重播手順並計算攻方形成死四、活四或四四的攻擊手。

## 2. N 點

N 點只分為兩種獨立標記：

- `NO_BLACK`：黑方不能使用。
- `NO_WHITE`：白方不能使用。

同一點可以同時具有兩種標記，表示黑白雙方都不能使用。

判斷時依準備放入棋子的顏色決定：

```text
放黑棋 → 檢查 NO_BLACK
放白棋 → 檢查 NO_WHITE
```

## 3. 第一版初始材料

第一版只使用連續活三：

```text
留 留 攻 攻 攻 留
```

- 三個「攻」為攻方棋子。
- 三個「留」同時標記 `NO_BLACK` 與 `NO_WHITE`。
- 第二個「留」是後續形成活四的攻擊點。
- 最外側兩個「留」是活四後的兩個五連點。
- 三顆攻方棋子都可作為 A。

初始死四與任意步數生成，等 N 點重算規則完成後再開放。

## 4. 新增死四模板

不考慮鏡射時共有三種：

```text
1. X 五 攻 攻 攻 攻 X
2. X 攻 五 攻 攻 攻 X
3. X 攻 攻 五 攻 攻 X
```

其中：

- `攻`：攻方棋子。
- `五`：攻方形成死四後，守方唯一被迫防守的位置。
- `X`：模板外側位置。
- A 必須是原盤面既有攻方棋子，並與模板中的任一 `攻` 位置重合。

## 5. 模板相容規則

### 5.1 攻方棋子位置

可使用：

- 原盤面已有的攻方棋子。
- 空點，且不是攻方 N 點；此時新增一顆攻方棋子。

不可使用：

- 守方棋子。
- 攻方 N 點上的空點。

### 5.2 A 點

A 必須：

- 原盤面已是攻方棋子。
- 與新死四模板中的一個 `攻` 位置重合。
- 完整套入死四後再拿掉，成為題目的第一手。

### 5.3 五點

五點：

- 不能是攻方 N 點。
- 不能已有攻方棋子。
- 可以是空點。
- 可以原本是守方棋子；生成時可先移除，VCF 過程中守方會下回該點。

### 5.4 X 點

X 可以是：

- 空點。
- 守方棋子。
- 任一種 N 點。
- 盤端。

唯一基本限制是不能已有攻方棋子。

### 5.5 移除 A 後的活三

移除 A 後，只檢查新死四方向是否形成合法活三。

若形成活三，只能在模板兩端 X 補守方棋子。可能需要：

- 不補。
- 補一側。
- 兩側都補。

若要在某個 X 新增守方棋子，該點不能是守方 N 點。

有禁手且黑方為攻方時，若 X 對黑方而言是禁點或長連失效點，該端可視為無效，不必補白。

## 6. 隨機生成

每次生成：

1. 隨機建立初始活三。
2. 從三顆既有攻方棋子中選 A。
3. 從其他方向套入三種死四模板之一。
4. 移除 A。
5. 必要時在 X 加入守方棋子。
6. 使用現有 VCF 引擎完整驗證。

候選以加權隨機排列，不直接選最高分：

- 沿用更多既有攻方棋子可有小幅加成。
- 朝天元方向生成可有小幅加成。
- 兩項加成都可關閉，且不得大到破壞隨機性。

不設定每層最大嘗試次數；使用者可隨時停止。

## 7. 第一版延伸驗證

第一版從活三產生 2 步 VCF，目標路線必須符合：

1. VCF 步數為 2。
2. 第一手為新 A。
3. 第二手為新死四的五點。
4. 第三手回到初始活三指定的活四形成點。
5. 若手順繼續，後續防守與五連點必須是初始活三兩端的保留點。

多步版將改用「標準完成盤面」比較：

- 比較黑白棋子最終位置，不比較落子順序。
- A 在 VCF 過程中下回、五點原有守方棋子被移除後又下回，都屬沿用舊棋子，不算差異。
- 新舊完成盤面的差異必須正好等於本次永久增加的棋子。

## 8. 其他 VCF 的處理

使用現有「多組 VCF」功能檢查：

1. 目標 VCF 必須成功延伸，否則放棄本次候選。
2. 若任何其他 VCF 比目標 VCF 短，放棄本次候選。
3. 等長或更長的其他 VCF 允許。
4. 不需要額外保存其他組的分支數或數量。

## 9. 非要求

- 不要求生成盤面可以由正常黑白交替對局走出來。
- 第一版不自動補白封鎖其他較短 VCF；遇到較短組直接淘汰候選。
- 第一版不使用 Worker Pool 平行驗證。

## 10. 後續項目

- 從現有 VCF 路線重新計算完整 N 點。
- 初始死四。
- 任意目標步數。
- 標準完成盤面比較與多層延伸。
- 必要時加入候選快取與平行驗證。
